## 📍 Contexte Global Akeli

### La Vision Centrale : Le Réalignment

Akeli n'est pas une application de régime. C'est une plateforme de **réalignment nutritionnel**.

**Le constat fondamental :**
- Nos modes de vie ont changé radicalement (sédentarité européenne vs vie active ancestrale)
- Nos cuisines traditionnelles n'ont pas suivi ce changement
- Il existe une **désynchronisation** entre ce qu'on mange et comment on vit
- Ce n'est pas la cuisine qui est le problème, c'est le **décalage**

**La promesse Akeli :**
> "Tu n'as pas à choisir entre qui tu es et comment tu veux te sentir."

### Le Modèle B2B2C Révolutionnaire

**L'insight stratégique :**
> "Akeli pense créateur en premier. L'utilisateur est la conséquence, pas le point de départ."

**Architecture du modèle :**
```
Curtis → Créateurs (message business universel)
    ↓
Créateurs → Leur niche (message authentique de vie)
    ↓
Niche → Utilisateurs Akeli (préqualifiés, engagés)
```

**Ce qui distingue Akeli :**
- Les créateurs ne sont PAS des influenceurs — ce sont des **actifs économiques réels**
- Rémunération à la **consommation nutritionnelle réelle**, pas aux vues/likes
- Revenu passif stable, indépendant des algorithmes
- Chaque créateur recruté = tête de pont dans un marché spécifique

### Les Personas Stratégiques

**Utilisateur cible (app mobile) : Fatoumata/Marianne, 40-45 ans**
- Femme sédentaire, classe moyenne, cuisine traditionnelle = identité
- **Invisible dans le discours nutrition dominant** (fitgirls, influenceuses)
- Alimentation = seul levier réel de changement
- Test ultime d'Akeli : si ça marche pour elle, ça marche pour tout le monde

**Créateur cible (website) : Aminata/Chidi, 25-35 ans**
- 1K-50K abonnés (micro/mid-tier en priorité)
- Cuisine culturelle authentique
- Audience engagée > audience large
- Cherche revenu stable indépendant des algorithmes

## 🎯 Identité du Website V1

### Ce que le Website N'est PAS

❌ **Pas un site marketing classique**  
Le website n'est pas une landing page pour télécharger l'app. Il ne parle pas aux utilisateurs finaux (consommateurs).

❌ **Pas un simple back-office**  
Le website n'est pas juste un tableau de bord technique. Il porte une vision, un message, une reconnaissance.

❌ **Pas une vitrine passive**  
Le website n'est pas un catalogue statique de recettes. C'est un espace de travail actif.

### Ce que le Website EST

✅ **Un outil professionnel pour créateurs**  
Le website est l'espace de travail où les créateurs construisent leur catalogue, gèrent leurs revenus, comprennent leur audience.

✅ **Un point d'entrée B2B pour recrutement**  
Le website est le lieu où un créateur potentiel découvre Akeli, comprend le modèle économique, et décide de rejoindre.

✅ **Une reconnaissance de valeur**  
Le website traite le savoir culinaire culturel comme un actif économique réel. Il reconnaît la valeur nutritionnelle, culturelle et émotionnelle des recettes.

✅ **Un pont entre créateur et audience**  
Via les profils publics créateurs, le website rend visible la relation créateur ↔ utilisateurs (consommations, Mode Fan, performance).

## 🧭 Rôle du Website dans l'Écosystème Akeli

### Relation Website ↔ App Mobile

**Séparation claire des audiences :**

| Surface | Audience | Fonction Principale |
|---------|----------|---------------------|
| **App Mobile** | Utilisateurs consommateurs (Fatoumata, Marianne) | Découverte recettes, meal planning, suivi nutrition, Mode Fan |
| **Website** | Créateurs (Aminata, Chidi) | Création recettes, gestion catalogue, revenus, analytics |

**Ponts entre les deux :**
- Profil public créateur (accessible web + app mobile)
- Deep links : app → profil créateur web (si besoin)
- Données partagées : recettes créées sur website → consommées dans app

**Principe :**
- Les **créateurs ne sont pas des utilisateurs de l'app mobile** (sauf s'ils le souhaitent personnellement)
- Les **utilisateurs ne créent pas de recettes** (ils consomment)
- Chaque surface a son audience, son langage, son UX spécifique

### Position dans le Flywheel B2B2C

```
1. RECRUTEMENT (Website landing page)
   Curtis recrute créateurs via lives TikTok
   → Landing page explique modèle économique
   → Signup créateur (auto-approved)

2. ACTIVATION (Website création recettes)
   Créateur publie 10-30 recettes
   → Interface rapide, faible friction
   → Recettes = actifs économiques visibles

3. MONÉTISATION (Website dashboard revenus)
   Créateur voit revenus passifs croître
   → Transparence totale sur consommations
   → Motivation à créer plus de contenu

4. ACQUISITION UTILISATEURS (App mobile)
   Créateur partage ses recettes Akeli à son audience (TikTok, Instagram)
   → Audience télécharge app pour suivre créateur
   → Utilisateurs préqualifiés, engagés

5. RÉTENTION (App mobile + Website)
   Utilisateurs consomment recettes créateur
   → Créateur voit consommations en temps réel (website)
   → Cercle vertueux : plus de consommations = plus de revenus = plus de création
```

## 💬 Message Créateur

### Ton de Communication

Le website parle **AUX créateurs**, pas DES créateurs.

**Ton général :**
- **Respectueux** : reconnaissance de la valeur de leur savoir
- **Professionnel** : business, revenus, analytics, données concrètes
- **Honnête** : transparence sur le modèle économique, pas de promesses irréalistes
- **Inclusif** : parle à tous les créateurs (micro à mid-tier), pas juste aux "influenceurs"

**Ce qu'on dit :**
- ✅ "Ton savoir culinaire a une valeur économique réelle"
- ✅ "Construis un revenu passif stable autour de ta passion"
- ✅ "Indépendant des algorithmes de TikTok et YouTube"
- ✅ "Rémunération à la consommation réelle, pas aux vues"

**Ce qu'on ne dit pas :**
- ❌ "Deviens riche rapidement"
- ❌ "Remplace ton salaire en 3 mois"
- ❌ "Rival de YouTube/TikTok"
- ❌ Jargon marketing creux

### Pitch Créateur par Marché

**Marché francophone (France, Sénégal, Côte d'Ivoire, Belgique) :**
> "Tes recettes africaines méritent d'être reconnues et rémunérées. Akeli est la première plateforme qui paie les créateurs à chaque repas consommé — pas au like, pas à la vue. Construis un revenu passif stable autour de ta passion."

**Marché anglophone (Nigeria, UK, US) :**
> "Tu cuisines, tu as une audience, tu veux un revenu récurrent indépendant des algorithmes de TikTok et YouTube. Akeli te donne deux leviers : un revenu passif à chaque consommation et la vente directe de tes programmes nutritionnels à ton audience. C'est ton contenu, c'est ton revenu."

### Transparence du Modèle Économique

**Principe :** Transparence totale dès la landing page.

Le website explique ouvertement :
- Modèle €1 pour 90 consommations (0,0111€/consommation)
- Lecture correcte : **audience active** > unités de consommation
- Exemple concret : "100 abonnés actifs = 100€/mois en revenu passif"
- Vente de programmes : "Créateur fixe le prix, conserve 90%, Akeli prend 10%"
- Mode Fan : "1€/mois par fan, allocation directe créateur"

**Pourquoi transparent ?**
- Filtre naturel : attire les créateurs sérieux, repousse les opportunistes
- Crédibilité : pas de promesses irréalistes
- Confiance : le créateur sait exactement ce qu'il gagne et pourquoi

## 🎨 Priorités Fonctionnelles (Basées sur la Philosophie)

### Principe de Priorisation

Chaque feature doit servir **un des trois piliers** :

1. **Faible barrière d'entrée** (faciliter la création, réduire friction)
2. **Transparence économique** (rendre revenus visibles, compréhensibles)
3. **Reconnaissance de valeur** (valoriser le savoir culinaire, célébrer succès)

### Priorité 1 — Création Recette (Wizard)

**Philosophie :**  
La création de recette est **l'acte central** du créateur sur Akeli. Elle doit être :
- Rapide (< 30 min pour première recette)
- Simple (pas besoin d'expertise technique)
- Valorisante (le créateur sent que son contenu a de la valeur)

**Features clés :**
- Wizard 6 steps guidé
- Auto-save (aucune perte de données)
- Traduction automatique (toutes langues avec override manuel)
- Calcul macros automatique (avec tolérance à l'approximation)
- Preview avant publication

**Message porté :**  
> "Ton savoir culinaire mérite d'être partagé. Nous le rendons accessible à des milliers de personnes."

### Priorité 2 — Communauté (Chat Créateurs Complet)

**Philosophie :**  
La communauté créateurs est **source d'innovation et d'intelligence collective**. Les échanges entre créateurs génèrent des idées, patterns et opportunités.

**Features clés V1 :**
- ✅ Chat privé créateur ↔ créateur (1-to-1)
- ✅ Groupes créateurs (créés par créateurs eux-mêmes)
- ✅ Chat support créateur ↔ équipe Akeli
- ✅ Partage images/liens dans discussions
- ✅ Notifications nouveaux messages
- ✅ Typing indicators, read receipts
- ✅ Recherche dans conversations
- ❌ **Pas de traduction IA messages** (créateurs utilisent langue commune)

**Technologie :** Supabase Realtime (WebSockets)

**Groupes :**
- **Pas de groupes pré-créés par équipe Akeli**
- Créateurs créent leurs propres groupes librement
- Auto-organisation communauté (émergence organique)
- Modération légère (IA détection spam/insultes)

**Langues dans chat :**
- Créateurs communiquent dans langue de leur choix
- Groupes s'auto-organisent par langue (ex: groupe francophones, anglophones, etc.)
- Pas de barrière technique imposée

**Message porté :**  
> "Chaque cerveau est une ressource. Ensemble, vous créez une intelligence collective."

**Note Intelligence Collective :**
- Les échanges créateurs = source d'innovation détectable par IA (V1.5+)
- Patterns émergents collectés passivement en V1
- Analysis active et redistribution insights en V2

**Timeline Impact :**
- Chat complet = feature complexe
- Estimation : **+3-4 semaines** vs chat simple
- **Timeline totale V1 : 10-12 semaines** (acceptable)

### Priorité 3 — Gestion Catalogue

**Philosophie :**  
Le catalogue de recettes est **un actif qui s'apprécie avec le temps**. Le créateur doit pouvoir le gérer facilement.

**Features clés :**
- Liste recettes (publiées + brouillons)
- Filtres (type repas, région, tags)
- Édition rapide
- Duplication recettes
- Dépublication temporaire (si besoin)
- Stats par recette (consommations, revenus générés)

**Message porté :**  
> "Ton catalogue est ton patrimoine. Gère-le comme un professionnel."

### Priorité 4 — Profil Public Créateur

**Philosophie :**  
Le profil public est **la vitrine du créateur**. Il rend visible sa valeur auprès de son audience et des utilisateurs Akeli.

**Features clés :**
- URL propre : `a-keli.com/creator/[username]`
- Avatar, bio, région, spécialités, réseaux sociaux
- Stats publiques (X recettes, Y repas consommés)
- Badge Mode Fan (si éligible ≥30 recettes)
- Grid recettes du créateur

**Message porté :**  
> "Tu n'es pas juste un créateur de contenu. Tu es un acteur économique reconnu."

### Priorité 5 — Dashboard Revenus (IA-Expliqué)

**Philosophie :**  
Les chiffres seuls intéressent peu. Ce qui compte, c'est **l'IA qui explique ce qu'ils signifient** et comment progresser.

**Features clés :**
- Revenus du mois (€) — **avec explication IA**
- Consommations totales — **avec contexte IA**
- Progression vers objectifs — **avec encouragements IA**
- Top 5 recettes — **avec analyse IA "pourquoi elles performent"**
- Graphique évolution — **avec insights IA "patterns détectés"**
- **Mode Fan stats** (si éligible ≥30 recettes)

**Bouton principal :** "Explique-moi mes stats" (IA génère rapport en langage naturel)

**Message porté :**  
> "Les chiffres ne parlent pas d'eux-mêmes. L'IA les transforme en actions concrètes pour toi."

**Exemple explication IA :**
```
"Ce mois : 45 consommations, 0.50€ gagné.

Analyse :
Ta recette 'Thiéboudienne' performe bien (+20% vs mois dernier).
Les utilisateurs l'aiment car :
- Rapide (30min)
- Portions famille
- Publiée dimanche soir (moment optimal)

Suggestion :
Crée 2-3 autres recettes familiales rapides.
À ce rythme, tu atteindras 1€ dans 15 jours.
Objectif 30 recettes (Mode Fan) : 18 recettes restantes."
```

### Priorité 6 — Mode Fan (V1)

**Philosophie :**  
Le Mode Fan permet aux utilisateurs de **soutenir directement leur créateur préféré** avec un abonnement dédié.

**Fonctionnement :**
- User paie **€1/mois** pour activer Mode Fan avec 1 créateur
- 90% des recettes de son plan viennent de ce créateur
- Créateur reçoit **€1/mois garanti** par fan
- Éligibilité créateur : **≥30 recettes publiées**

**Features dashboard créateur :**
- Badge "Mode Fan Activé" sur profil public (si ≥30 recettes)
- Stats Mode Fan :
  - Nombre de fans actuels
  - Revenus Mode Fan ce mois
  - Evolution fans (graphique)
- Message incitatif si < 30 recettes :
  ```
  "Publie encore X recettes pour activer Mode Fan
   et recevoir des revenus garantis de tes fans !"
  ```

**Impact business :**
- Revenus prévisibles pour créateurs (fans = abonnements récurrents)
- Fidélisation utilisateurs (engagement créateur spécifique)
- Motivation créateurs (objectif 30 recettes concret)

**Message porté :**  
> "Tes fans les plus fidèles peuvent te soutenir directement. Chaque fan = 1€/mois garanti."

## 🎯 Design & UX Principles

### Design System (Cohérence avec App Mobile)

**Couleurs :**
- Primaire : `#9c88ff` (Violet)
- Secondaire : `#3bb78f` (Vert)
- Accent : `#FF9F1C` (Orange)
- Background : `#f1f4f8`
- Card : `#ffffff`

**Typography :**
- Font : Inter (Google Fonts)
- Headings : 32-36px, font-weight 700
- Body : 16px

**Components :**
- Tailwind CSS + shadcn/ui
- Cards : border-radius 12px, shadow subtle
- Sidebar : 280px fixe (navigation créateur)

### UX Principles

**1. Clarté > Complexité**  
Chaque page a un objectif clair. Pas de surcharge cognitive.

**2. Feedback immédiat**  
Chaque action (save, publish, edit) a un retour visuel immédiat.

**3. Progressivité**  
Les features avancées se révèlent progressivement. Pas d'overwhelm au début.

**4. Célébration des succès**  
Première recette, premier euro, 10 recettes, Mode Fan éligible → célébré visuellement.

**5. Transparence totale**  
Aucun chiffre caché. Revenus, consommations, stats → toujours visibles.
